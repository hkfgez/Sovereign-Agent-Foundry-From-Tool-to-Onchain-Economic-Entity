# Submission Checklist

This file is used to keep the final submission clean, complete, and easy to review.

The goal is not just to submit a repository.

The goal is to submit a coherent product narrative with clear visual proof.

---

## Core Submission Materials

### 1. X Article
Must clearly explain:

- what Sovereign Agent Foundry is
- what problem it solves
- why sovereign agents matter
- how the system works
- what was built in the demo
- why this is relevant to agent economy infrastructure

### 2. Demo Video
The video should show the full system in order:

- create the core agent
- initialize treasury
- activate market layer
- hire sub-agents
- open proof dashboard

Recommended duration:

- 45 seconds to 90 seconds

### 3. Repository
The repository should include:

- README
- architecture documentation
- demo script
- environment example
- page structure
- mockable modules or scripts

### 4. Architecture Diagram
A single clean diagram is enough.

It should show:

- core agent
- identity and wallet layer
- treasury layer
- market layer
- sub-agent layer
- proof dashboard

### 5. Screenshots
Recommended screenshots:

- homepage
- create agent page
- treasury setup section
- sub-agent allocation section
- dashboard proof view

---

## Product Messaging Checklist

Before final submission, confirm the project is presented as:

- not a chatbot
- not a generic AI wrapper
- not only a trading bot
- not a gimmick token launcher

It should be presented as:

- a sovereign agent framework
- an onchain economic structure
- a modular agent institution prototype
- a proof-oriented agent economy system

---

## Core Concepts That Must Be Visible

The final submission should make these ideas easy to spot:

### Identity
The agent has a persistent role and an addressable structure.

### Treasury
The agent owns and allocates capital.

### Market Layer
The agent has an economic surface visible to the market.

### A2A Collaboration
The agent can fund and coordinate sub-agents.

### Proof
The system exposes visible evidence of actions and capital movement.

---

## Demo Quality Checklist

Before recording the final video, verify:

- the UI text is readable
- the flow is short and linear
- there is no dead page in the demo
- the dashboard contains believable metrics
- the treasury movement is visible
- the sub-agent roles are easy to understand

---

## Repository Quality Checklist

Before publishing the repository, verify:

- README is clean and complete
- folder structure is readable
- no broken placeholder text remains
- .env.example is included
- docs are consistent with the UI
- file names are clear and professional

---

## Narrative Quality Checklist

Before publishing the X Article, verify:

- the opening hook is strong
- the project sounds like infrastructure, not hype
- the explanation is understandable in one pass
- the project thesis is consistent from beginning to end
- the closing line feels definitive

---

## Final Reminder

The strongest version of this project is not:

“an agent that does things.”

It is:

“an agent that can exist, own, allocate, collaborate, and prove itself onchain.”
