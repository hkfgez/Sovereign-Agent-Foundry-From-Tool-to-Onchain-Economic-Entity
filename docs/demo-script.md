# Demo Script

This demo is designed to communicate one core idea:

An AI agent can become a structured onchain economic entity.

The full demo should stay short, clear, and visual.

---

## Demo Goal

By the end of the video, the viewer should understand that the agent has:

- identity
- wallet
- treasury
- market layer
- worker agents
- proof of economic activity

---

## Step 1 — Create the Core Agent

Open the app and create the primary agent.

Show:

- agent name
- role
- mission
- creation confirmation

Suggested narration:

> We begin by creating the primary sovereign agent.  
> This is not a disposable bot instance.  
> It is the core economic actor of the system.

---

## Step 2 — Initialize Treasury

Move to treasury setup.

Show:

- treasury initialization
- starting balance
- budget policy
- reserve threshold

Suggested narration:

> The agent now receives an initial treasury.  
> This gives it operational capital and turns it from software into a fundable entity.

---

## Step 3 — Activate Market Layer

Open the market module.

Show:

- token metadata
- market activation state
- launch confirmation

Suggested narration:

> Next, the agent is given a market-facing layer.  
> It is no longer only operational.  
> It is now economically addressable.

---

## Step 4 — Hire Sub-Agents

Create two worker agents and allocate budget.

Recommended worker roles:

- Scout Agent
- Treasury Monitor Agent

Show:

- worker names
- assigned role
- budget allocation
- transfer record

Suggested narration:

> The core agent now hires two specialized sub-agents.  
> Capital is allocated from the core treasury to operational workers.

---

## Step 5 — Display Proof Dashboard

Open the dashboard.

Show:

- treasury balance
- outgoing allocations
- recent agent actions
- active workers
- wallet flow summary

Suggested narration:

> Finally, the proof dashboard shows the system as an auditable economic structure.  
> Capital moved, agents acted, and the activity can be inspected.

---

## Closing Line

Suggested closing line:

> We did not build another agent tool.  
> We built the minimum viable institution for a sovereign onchain agent.

---

## Recording Notes

Keep the pace steady.

Do not over-explain every UI action.  
Let the structure speak for itself.

The best demo tone is calm, confident, and minimal.

The project should feel inevitable, not theatrical.
