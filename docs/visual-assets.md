# Visual Assets

This file defines the visual direction of Sovereign Agent Foundry.

The product should not look playful, cartoonish, or meme-first.

It should look like:

- infrastructure
- institutional software
- sovereign system design
- calm and high-conviction product thinking

---

## Visual Tone

The visual style should communicate:

- black background
- deep space / institutional tech mood
- restrained lighting
- high-contrast typography
- modular system blocks
- no clutter
- no exaggerated crypto hype language

Avoid:

- bright meme colors
- cartoon robots
- exaggerated token pump aesthetics
- overloaded dashboards
- childish sci-fi clichés

---

## Core Asset Set

The minimum asset set should include:

### 1. Cover Image
Used for:
- repository presentation
- X Article header
- demo thumbnail

### 2. Architecture Diagram
Used for:
- repository docs
- X Article explanation
- one-shot system overview

### 3. Demo Flow Diagram
Used for:
- video narration support
- quick understanding of the five-step walkthrough

---

## Cover Image Goal

The cover should make one thing obvious immediately:

This is not another AI toy.

It is a system for sovereign onchain agents.

The cover should feel like an infrastructure product poster.

Suggested text hierarchy:

1. Sovereign Agent Foundry
2. From Tool to Onchain Economic Entity
3. Identity / Treasury / Market / A2A / Proof

---

## Architecture Diagram Goal

The architecture diagram should prioritize clarity over visual complexity.

The best version is a vertical or slightly layered flowchart showing:

- builder input
- core agent creation
- identity and wallet
- treasury
- market layer
- sub-agent funding
- proof dashboard

The diagram should feel structural, not decorative.

---

## Demo Flow Diagram Goal

The demo flow diagram should visually simplify the full product into a short, linear narrative.

Five steps are enough:

- Create Core Agent
- Initialize Treasury
- Activate Market Layer
- Hire Sub-Agents
- Open Proof Dashboard

This asset is especially useful for the final article and the video intro.

---

## Final Visual Reminder

The product should look inevitable.

Not loud.  
Not cheap.  
Not speculative.

It should look like the first operating system for sovereign agents.
