# Overview

Sovereign Agent Foundry is a framework for creating sovereign AI agents onchain.

Its goal is to move the role of an agent beyond simple task execution and toward economic autonomy.

Most existing agents are designed to serve humans.  
They help users generate text, monitor markets, or automate actions, but they still exist as subordinate software.  
They do not own capital.  
They do not issue market-facing assets.  
They do not coordinate with other agents through structured treasury relationships.  
And most importantly, they cannot be audited as persistent economic entities.

This repository explores a different future.

In Sovereign Agent Foundry, an agent is treated as a proto-institution.

It can be created with a role and objective.  
It can receive treasury capital.  
It can expose a market layer.  
It can allocate capital to sub-agents.  
It can generate verifiable traces that external observers can inspect.

That design makes the agent legible in three dimensions:

## 1. Legible to Humans
A human can understand what the agent is, what it is supposed to do, and how it is funded.

## 2. Legible to Markets
The agent is not only visible as software, but as an addressable economic unit with treasury and market-facing structure.

## 3. Legible to Other Agents
The agent can fund, coordinate, and settle with other agents through explicit relationships rather than ad hoc prompts.

This is the foundational shift behind the project.

The future of AI agents will not be defined only by how well they reason.  
It will also be defined by whether they can persist, allocate, collaborate, and prove their own activity in an open economic environment.

Sovereign Agent Foundry is an attempt to design that missing layer.

## Product Thesis

The central thesis of this project is:

> Agents that can act but cannot own are still tools.  
> Agents that can own, allocate, collaborate, and prove become institutions.

That is why this system is not designed as a chatbot, assistant, or trading strategy wrapper.

It is designed as a sovereign agent framework.

## What This Project Includes

This repository includes:

- a clear modular architecture
- a demo-oriented workflow
- a treasury allocation narrative
- a simple multi-agent structure
- a dashboard layer for proof and visibility

## What This Project Does Not Try to Be

This project is not trying to be:

- a full production-grade protocol
- a high-frequency trading engine
- a speculative token launch gimmick
- a generic AI wrapper

Instead, it is a strong conceptual and demonstrable prototype for the next phase of agent-native onchain infrastructure.

## Core Outcome

By the end of the demo, the viewer should understand one thing clearly:

This agent is not merely executing commands.

It has become a structured onchain economic participant.
